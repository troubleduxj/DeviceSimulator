# 系统监控与数据手册

本手册指导您如何监控模拟器的运行状况并分析生成的数据。

## 1. 系统监控 (System Monitor)
**系统监控** 仪表板提供了服务器性能和状态的概览。

### 1.1 资源使用情况
- **CPU 与内存**: 实时图表显示模拟器后端的资源消耗。
- **磁盘使用**: 可用存储空间。

### 1.2 服务状态
- 检查关键组件的连接性：
   - **后端 API**
   - **TDengine 数据库**
   - **MQTT 代理** (如果已启用)
   - **Modbus 服务** (如果已启用)

## 2. TDengine 数据查看器
**TDengine 查看器** 允许您检查存储在时序数据库中的原始数据。

1. **选择表**: 选择超级表（分类）或特定子表（设备）。
2. **时间范围**: 按时间过滤数据。
3. **数据网格**: 以表格格式查看带有时间戳的指标数据。
4. **刷新**: 手动获取最新记录。

## 3. 系统日志与 AI 分析
**日志查看器** 显示仿真引擎的操作日志。

### 3.1 查看日志
- 日志按严重程度进行颜色编码：
   - **Info (信息)**: 一般事件（蓝色）。
   - **Warning (警告)**: 潜在问题（黄色）。
   - **Error (错误)**: 严重故障（红色）。
- **筛选**: 使用搜索栏查找特定事件。

### 3.2 AI 智能分析
模拟器集成 Gemini AI 来分析日志模式。
1. 点击 **"AI 分析"** 按钮（闪烁图标）。
2. AI 将读取最近的日志并生成报告，内容包括：
   - **摘要**: 最近发生了什么。
   - **根本原因分析**: 为什么会发生错误（例如，“设备 01 报告过热，原因为应用了‘冷却液泄漏’场景”）。
   - **建议**: 建议采取的行动。
3. 结果以格式化的 Markdown 视图显示，并可复制到剪贴板。
